🦆 DuckDB query workbench

A real analyst workflow: query remote Parquet with DuckDB-WASM in your browser. Each query pulls only the bytes it needs over HTTP range requests — run the same one through direct S3, the proxy, and the PR #189 edge cache, and compare.

  • DuckDB reads remote Parquet with HTTP range requests — the query time is dominated by how fast those range reads come back. ↓ lower is better. Fastest endpoint highlighted.
  • The result is identical across endpoints (same object) — only latency differs. The comparison summary pits the edge cache against direct S3 and the no-cache proxy.
  • The first run loads DuckDB-WASM (a few MB) — that one-off cost isn't the endpoint's. Run a query twice: the second is the fair, warm comparison. Judge ratios, not absolute ms.

DuckDB-WASM is loaded from jsDelivr (single-thread build — no cross-origin isolation needed). Object keys are identical across endpoints. Source: proxy-perf-demo.
